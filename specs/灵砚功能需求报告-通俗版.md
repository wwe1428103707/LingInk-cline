## 模块 A：核心写作辅助

### A1. 选题引导

**功能目的**：在选题和写作初期，通过苏格拉底式追问帮作者检验选题价值、逼出清晰思路，并提供 AI 辅助回答作为备选。

**应用场景**：开题方向不确定、面对空白文档思路模糊、想确认选题是否值得做。

**实现要求**：
1. 用户输入选题或模糊主题，AI 依次追问三个核心问题：
   - **边界之问**：时间、空间、对象的边界在哪里？
   - **贡献之问**：哪个已有结论会被修正或补充？
   - **可行之问**：现有数据/资料/方法能否支撑论证？
2. 若用户在某轮回答困难，可点击"求助 AI"——AI 生成参考回答供用户判断修改
3. 三轮后生成"选题单"，包含选题陈述+三问三答，可直接发给导师讨论
4. 进阶模式：持续 5~10 轮苏格拉底反问，从具体场景→本质差异→可验证命题，生成"思维路径实录"+开头段草稿

**示例**：
> 用户："我想研究短视频对青少年心理健康的影响。" AI 追问时间/空间/对象边界，用户答不上来→点"求助 AI"→AI 给出"建议边界：2019-2024年中国大陆13-18岁青少年"→用户确认或修改→继续追问贡献之问→最终生成"选题单"导出

---

### A2. 结构重组

**功能目的**：同一内容可按不同叙事逻辑组织，支持全篇或选中部分段落重新编排结构。

**应用场景**：草稿完成后对结构不满意但不知如何调整；写作中途想预判局部段落的叙事逻辑是否合理。

**实现要求**：
1. 支持两种分析模式：
   - **全篇模式**：草稿完成后自动提取全文内容，生成三种叙事路径大纲
   - **局部模式**：写作中随时选中若干段落/章节，生成被选中部分的叙事预览
2. 三种叙事路径：
   - **问题导向型**：先抛出问题再层层解谜，适合创新性强的研究
   - **方法驱动型**：强调方法突破再展示应用，适合方法论文章
   - **争议切入型**：从既有争议起手逐步给出方案，适合综述或理论文章
3. 每种路径给出重新编排的章节顺序建议及"为什么这样更好"

**示例**：
> 用户写完方法与初步结果，选中这两章点击"结构重组"→AI 生成两种叙事路径预览，帮用户判断是先展示全部方法再展示全部结果，还是逐项方法-结果交替

---

### A3. 概念溯源

**功能目的**：帮助作者准确界定核心概念，避免术语使用模糊导致的学术硬伤。

**应用场景**：写核心概念定义时不确定如何界定；切换学派视角时需澄清术语。

**实现要求**：
1. 用户在论文中右键点击一个术语，选择"概念溯源"
2. AI 通过学术数据库摘要（Crossref / Semantic Scholar 等）抓取该术语的主流学术定义
3. 列出不同学派对该概念的不同界定（含出处），并标注"你目前使用更接近 X 学派的用法，是否需要在文中明确？"
4. **补充方案**：支持连接用户本地 Zotero/Mendeley 数据库作为检索源，降低对在线 API 的依赖
5. 生成一段可直接嵌入论文的"概念界定"草稿（含引用）

**示例**：
> 用户右击"社会资本"，选择"概念溯源"。AI 返回：Bourdieu(1986) 定义为"实际或潜在资源的集合……"；Putnam(2000) 定义为"社会组织的特征，如信任、规范和网络……"；Lin(2001) 从个体层面定义为"嵌入在社交网络中的资源……"。AI 判断："你文中的使用接近 Bourdieu 的场域视角，建议在脚注中明确。是否需要生成一段概念界定？"

## 模块 B：AI 智能体与记忆

*本模块功能依赖 Hermes Agent 核心——具备规划、工具调用、推理链和长期记忆能力的本地 AI 智能体。设计目标：在单用户桌面环境中实现高自主性的学术辅助。*

### B1. 投稿匹配与格式

**功能目的**：帮助研究者找到最适合的投稿期刊，并自动完成格式整备，节省反复试投的时间。

**应用场景**：论文定稿后不确定投哪本期刊；格式调整繁琐费时。

**实现要求**：
1. Agent 提取论文特征：主题、方法、引用特征、创新程度
2. 调用期刊数据库 API 检索匹配期刊，按多维度排序：主题匹配度、影响因子、审稿周期、接收率
3. 生成推荐列表（含推荐理由）
4. 用户选定目标期刊后，Agent 自动调用 Pandoc 进行格式转换（符合该刊模板）
5. 执行前需用户确认，支持"自动模式"（关键节点暂停）和"半自动模式"（每步需审批）

**示例**：
> 用户将一篇"社交媒体与大学生心理健康"的实证论文导入灵砚，点击"投稿匹配"。Agent 列出《Journal of Youth Studies》《New Media & Society》《Computers in Human Behavior》并标注各刊审稿周期和接收率。用户选定后者后 Agent 自动完成格式转换和引用风格调整。

---

### B2. 文献检索综述

**功能目的**：将"找文献→读摘要→分类→写综述"的繁琐流程压缩为 Agent 驱动的自动化流水线。

**应用场景**：写文献综述章节时，每次都要重新搜索、筛选、阅读、撰写。

**实现要求**：
1. 用户输入研究主题和检索范围（时间、语言、数据库）
2. Agent 生成多轮检索策略（关键词组合、同义词扩展、引用追踪），自动调用学术 API 检索
3. 去重、筛选高相关文献，按主题自动聚类
4. 为每个主题簇生成综述段落草稿（含文献间对话关系，而非简单罗列）
5. **增强**：支持两种检索源——在线学术 API（默认）+ 用户本地文献数据库（Zotero/Mendeley），可切换或混合使用
6. 用户可逐段确认、修改、插入到论文中

**示例**：
> 用户输入："检索 2020-2025 年关于 AI 辅助学术写作的英文文献，关注伦理争议和效果评估。"Agent 执行多轮检索，筛选 38 篇高相关文献，自动聚类为"AI 学术伦理""AI 润色效果""AI 对原创性的影响"三主题，每主题生成综述草稿。

---

### B3. 漏洞扫描

**功能目的**：在投稿前系统性地扫描论文论证漏洞，定位作者自己看不见的逻辑风险。

**应用场景**：论文定稿前最后的质量把关。

**实现要求**：
1. Agent 扫描全篇，按三个维度执行：
   - **前提检查**：每个主张所需的前提是否已论证？未论证的列为"隐藏假设"
   - **证据审查**：每个证据的强度是否匹配主张的力度？"强烈主张需强证据，此处只有描述性统计"
   - **反驳演练**：为每个核心论点生成 2 个最可能的反驳
2. 漏洞按高/中/低三级风险评级，高风险点必须处理
3. **用户可调严格度**——温和（仅标记高风险）/ 标准（高+中风险）/ 严格（全部+额外可疑项）
4. 输出《论证体检报告》，每个漏洞附带修改建议

**示例**：
> 扫描后《论证体检报告》显示：【高风险】第 3.2 节主张"社交媒体导致焦虑上升"，证据仅来自相关性分析，未控制"人格特质"混淆变量；【中风险】第 2 章假设"所有用户以相同方式使用平台"，文献中存在"主动 vs. 被动使用"区分，建议补充辨析。

---

### B4. 模拟审稿

**功能目的**：在投稿之前，从多个审稿人视角全方位审视论文，预判审稿意见。

**应用场景**：定稿后、投稿前的最后自查。

**实现要求**：
1. Agent 同时扮演 4 种角色对论文进行审稿：
   - **方法论专家**：样本量如何确定？为何选此方法而非替代方法？
   - **领域专家**：对某理论的引用是否过时？近三年有无更新的修正？
   - **统计审稿人**：报告的 p 值是否完整？是否遗漏效应量报告？
   - **挑剔读者**：过渡是否流畅？读者能否看懂从 A 到 B 的逻辑？
2. 生成《多角色审稿意见汇编》
3. 回应机制：可逐条回复，Agent 判断回应是否充分

**示例**：
> 一篇问卷调查论文，四角色分别提出：方法论专家问"样本量 200 如何确定充足性？"；领域专家指出"引用的量表已被 2022 年修订"；统计审稿人指出"缺效应量"；挑剔读者说"文献综述到假设仅一句话，跳跃"。

---

### B5. 写作偏好学习

**功能目的**：让 Agent 长期学习用户的写作习惯与偏好，越用越懂用户；同时集成写作风格分析与报告。

**应用场景**：每次使用 AI 辅助时不想重新描述风格偏好；想了解自己的写作弱点和演变轨迹。

**实现要求**：
1. Agent 在本地 SQLite + LanceDB 中持续学习：
   - **写作风格**：句长偏好、术语密度、被动/主动语态比
   - **论证习惯**："先立论后反驳"还是"逐一推进"的结构偏好
   - **引用偏好**：常引的期刊、作者、年代倾向
   - **审阅历史**：最常驳回的建议类型（下次少提此类建议）
2. **用户可控**：可查看/编辑/删除/暂停偏好学习
3. 定期生成"写作风格报告"：
   - **句式分析**：句式长度分布、用词偏好、论述密度
   - **高频问题**：最常被 AI 标记的问题类型 Top 5
   - **写作轨迹**：过去一年的写作习惯变迁——高产月、高效时段
4. 所有数据完全本地存储

**示例**：
> 用户使用灵砚三个月后，Agent 学到他偏好简洁短句（平均句长 18 词）、习惯在每段首句亮明观点、常用 APA 第 7 版、善于接受措辞建议但几乎不采纳段落重写建议。年度写作报告显示高产月为 3 月和 9 月，写作高峰时段 14:00-16:00。

---

### B6. 项目记忆

**功能目的**：让 Agent 记住每篇论文的背景信息，跨会话保持上下文，不需要每次对话都重新交代。

**应用场景**：论文写作周期长，每次打开灵砚都要回顾"上次做到哪了"。

**实现要求**：
1. 每个论文项目在本地维护一个项目记忆体（JSON 结构），记录：选题来源、核心假设、目标期刊、数据路径、最近会话讨论焦点、未完成的 Agent 任务、已检索文献/草稿片段
2. 用户问"上次那个建议怎么改？"Agent 能精确回溯到第几段的哪条建议
3. 打开论文时 Agent 自动回顾上次会话的关键决策和未完成事项
4. **用户可控**：可查看/编辑/删除项目元数据，或选择不开启项目记忆

**示例**：
> 用户周一让 Agent 找到 5 篇关于"短视频成瘾"的文献，周三打开灵砚时 Agent 自动提示："周一共检索到 5 篇相关文献，3 篇已标记为'相关'，2 篇待读。要继续文献综述的撰写吗？上次停在第 3 段。"

---

### B7. 工作流

**功能目的**：提供预设的工作流模板库，快速扩展 Agent 自动化能力。

**应用场景**：需要自动化执行重复任务（如每周文献速递、投稿前检查清单）但不想自己编排工作流。

**实现要求**：
1. 提供预设工作流模板库，如"每周文献速递""投稿前检查清单"等
2. Agent 按设定频率自动执行
3. 结果推送至灵砚面板
4. 每次执行日志可查看，支持暂停/修改/删除

**示例**：
> 用户在引导下配置"投稿前检查清单"模板，Agent 自动按序执行：检查引用格式→检查图表编号→检查章节标题层级→检查摘要关键词→生成检查报告标注不合规项。

---

### B8. 深度追问

**功能目的**：A1"选题引导"的 Agent 驱动升级版——跨会话、有状态记忆的深度推理追问。

**应用场景**：研究思路混沌期、理论构建困难、需要持续多轮思辨。

**实现要求**：
1. Agent 维持推理状态机，记住前 5 轮用户的答案，问题层层递进
2. 用户回答后 Agent 分析未明确的前提和矛盾点，追问更深层的问题
3. 对话结束后生成思维路径图+初稿骨架
4. 跨会话记忆：今天讨论的内容明天打开灵砚可以继续

**示例**：
> 用户昨天和 Agent 讨论了"数字劳工异化"的概念界定，今天打开灵砚，Agent 提示："上次我们谈到算法控制与传统异化的本质区别。你提到过想从'时间自主权悖论'切入，要继续从这个角度深入吗？"

## 模块 C：论证与逻辑深度

### C1. 论证链路图

**功能目的**：将论文的论证结构以可视化骨架图呈现，让作者一眼看到论证链条的完整性和断裂点。

**应用场景**：草稿完成后自我审查；导师反馈"逻辑跳跃"但不知具体哪里跳跃；写作中随时检查局部逻辑。

**实现要求**：
1. AI 自动提取每个段落的核心主张、证据类型、推理关系
2. 生成"论证链骨架图"：A → B → C 的节点关卡图，每个节点可折叠展开查看原文依据
3. 断裂处用虚线和红点标注并说明缺少什么
4. 点击断裂处，AI 生成 2~3 个可能的论证路径（每条标注所需补充的证据类型或逻辑步骤），用户选择后生成补全段落草稿
5. **增强**：支持全局分析（草稿完成后）+ 局部分析（写作中随时选中若干段落生成局部论证骨架图）

**示例**：
> 一篇论文的论证骨架显示：A（短视频使用时长增加）→ B（注意力分散）→**断裂**→ C（学业成绩下降）。点击断裂处，AI 给出三条补全路径：补充中介链条、补充认知机制、使用元分析文献过渡。

---

### C2. 假设检测

**功能目的**：挖掘论文中暗含但未论证的假设，标记逻辑风险，帮作者提前堵住审稿人可能攻击的逻辑漏洞。

**应用场景**：理论构建密集的论文（社科、人文）；论证环节较多的论文。

**实现要求**：
1. 用户选定一个段落（或全篇），AI 反问"这个结论成立的前提是什么？"
2. AI 列出该段落暗中依赖的 3~5 个假设，三分类标注：
   - **已论证**：在文中某处已有充分论证
   - **可引用**：属于领域共识，可直接引用某文献支撑
   - **未论证**：尚未论证，构成逻辑风险
3. 针对"未论证"假设，AI 提供"你可以这样补一笔"的开头句，或"建议引用以下文献"的检索建议
4. 可与 B3"漏洞扫描"联动——本功能挖掘隐含假设，B3 扫描已表述的论证漏洞

**示例**：
> 用户选定结论段"远程办公显著提升了员工工作满意度（β=0.34, p<0.01）"。AI 挖掘 4 个隐含假设：假设 1 已论证（匿名填写保障）；假设 2 可引用（Locke 定义文献）；假设 3 未论证（风险：统计显著≠实际显著，建议补充效应量解释）。

---

### C3. 反驳演练

**功能目的**：让 AI 扮演持相反立场的严谨学者，系统性反驳论文核心论证，通过"攻防演练"强化论证韧性。

**应用场景**：论文核心论证完成后；准备答辩时。

**实现要求**：
1. 用户选定核心章节，点击"反驳演练"
2. AI 扮演持相反立场的学者，生成 3 条最尖锐的反驳，并指出论证中最薄弱的环节
3. 用户逐条回应，AI 判断回应是否有力（给出评分和具体理由）
4. 直到所有反驳得到充分回应

> **注意**：本功能聚焦"主动反驳训练"，B3"漏洞扫描"聚焦"系统性漏洞扫描"，两者互补不合并。

**示例**：
> 用户核心论点："平台算法对外卖骑手的控制导致劳动异化。"反方生成三条反驳：(1) 骑手对弹性工时的偏好是主动选择而非被动异化；(2) 访谈样本仅来自单一城市的 15 名骑手；(3) 异化理论来自工业资本主义语境需理论修正。

## 模块 D：文献与知识管理

### D1. 知识补漏

**功能目的**：AI 在后台持续研读论文草稿，实时识别文献缺口、方法遗漏和理论盲区，从被动搜索变成主动情报。

**应用场景**：写作中不确定是否有遗漏的关键文献或更优方法。

**实现要求**：
1. AI 后台持续分析论文草稿，识别三类缺口：
   - **文献缺口**：核心论证未处理近两年被引最高的相反观点
   - **方法链**：方法的标准出处是否已正确引用
   - **数据支撑**：描述性统计后是否缺少必要的检验方法
2. 点击即插入建议，插入即自动格式化引用
3. **增强**：支持从用户本地文献库（Zotero/Mendeley）检索补充文献，降低对在线 API 的依赖

**示例**：
> AI 后台提示："你引用了 Anderson(2019) 的假新闻传播理论，但 Smith(2022) 在 Nature Human Behaviour 上对该理论提出了重要修正。建议引用 Smith 以展示文献覆盖完整度。是否需要找出这篇论文？"

## 模块 E：数据、代码与统计辅助

### E1. 图文校验

**功能目的**：跨越文字和视觉模态，让论文中的图表和文字描述相互验证、自动互译，减少图表表述不当。

**应用场景**：论文中的图表描述写不好；不确定图表是否支持文字论点。

**实现要求**：
1. **图转文**：上传数据图表（截图或图片），AI 自动生成标准学术图表描述并编号
2. **文转图**：选中一段描述数据的文字，AI 判断"此处理应配图"，生成图表草稿结构描述和示例数据点（可导出至 Excel/Python 精确绘制）
3. **图表一致性检查**：自动比对文中所有图表与正文描述，标注数值不一致处

**示例**：
> 用户上传柱状图截图，AI 生成："图 3：不同年龄组对短视频平台的日均使用时长（M ± SD）。18-25 岁组（n=120）使用时长显著高于 36-45 岁组（n=98），p<0.001。" 一致性检查发现："正文第 3.2 段写 18-25 岁组日均 127 分钟，但图 3 显示约 115 分钟，请核实。"

---

### E2. 结果表述（可选模块）

**功能目的**：将统计结果自动转换为符合学术规范的文字表述，并审核用户对统计结果的解读是否过度推论。

**应用场景**：非统计学专业作者写结果部分，统计术语使用不当或解释过度。

**实现要求**：
1. 用户粘贴统计软件输出（SPSS/R/Python 输出表格或关键数值），AI 做两件事：
   - **生成学术表述**：将 "p=0.032" 转化为 "差异具有统计学意义（p=0.032），表明……"
   - **解读审核**：检查用户对结果的解读是否过度推论
2. 检测到不当表述时温和提示并给出修改建议
3. **设为可选模块**，用户可在设置中开启/关闭

**示例**：
> 用户写"实验证明两组差异显著（p=0.03）"，AI 提示："建议改为'实验组与对照组之间存在具有统计学意义的差异（p=0.032, Cohen's d=0.45）'——使用'demonstrate'而非'prove'；r=0.32 解释 10% 方差比'强相关'更准确。"

---

### E3. 代码辅助（可选模块）

**功能目的**：面向计算机科学/数据科学等需在论文中嵌入代码的学科，提供代码到学术表达的多项转换工具。

**应用场景**：CS/工程论文需展示算法流程、代码注释、运行结果描述。

**实现要求**：
1. 默认关闭，需手动开启
2. 核心功能：
   - **伪代码提炼**：选中代码块（Python/R/MATLAB/JavaScript），AI 去除语言细节，提炼为 LaTeX（algorithm2e 格式）学术伪代码
   - **代码-论文一致性校验**：对比代码与论文方法描述的数值和结构一致性
   - **运行结果转学术表述**：从终端输出提取关键数值生成"结果"段落
   - **学术注释生成**：将代码注释从口语化提升为三版本（学术版/标准注释版/教学版）
3. 代码执行在沙箱中完成，确保安全

**示例**：
> 用户粘贴一段 40 行 Python 代码实现 K-means 聚类。AI 提炼为 10 行 LaTeX 伪代码并生成描述："算法 1 描述了基于欧氏距离的 K-means 聚类过程，迭代收敛条件为质心移动距离小于阈值 ε。"

## 模块 F：语言润色

### F1. 英文润色

**功能目的**：为非英语母语者提供从地道度校验到学术语用统一的完整润色方案。

**应用场景**：英文论文定稿前的语言精修；导师评语为"awkward"或"Chinglish"；需要确保全篇学术风格统一。

**实现要求**：
1. **基础层（地道度校验）**：按三级标记非地道表达
   - 轻度：可接受但不够简练（如 "in order to" → "to"）
   - 中度：略显生硬（如 "make a research" → "conduct research"）
   - 重度：可能引起误解（如 "it is obvious that" 在学术写作中暗示不严谨）
   - 每条建议附带"语感解析"解释为什么改后更自然
   - 按章节输出"语感健康度"评分
2. **进阶层（学术语用润色）**：三层递进加工
   - 精准化：修正术语、固定搭配
   - 得体性：调整论述力度和人际功能（如 "Obviously, this proves..."→"Taken together, these findings suggest..."）
   - 声调统一：确保全篇正式程度、被动/主动语态比、第一人称使用规则一致
   - 生成"声调地图"雷达图（正式度/确定性/亲和度/术语密度）
3. 用户可选仅基础层或两层全开

**示例**：
> 基础层：原文 "The results obviously prove that the hypothesis is correct." → 标记【重度】→ 建议改为 "These findings provide strong evidence supporting the hypothesis." → 语感解析："在英文学术写作中，'obviously' 和 'prove' 被视为过度断言。科学证据通常'支持'而非'证明'假设。"
> 进阶层：声调地图显示引言部分"确定性"偏高（过多绝对化断言）、讨论部分"亲和度"偏高（过多 "we believe"），AI 建议统一为"高正式度+中确定性+低亲和度"全篇声调标准。

## 模块 G：写作体验

---

### G2. 反馈风格切换

**功能目的**：让用户选择 AI 反馈的语气风格，在不同写作阶段匹配不同"人格"。

**应用场景**：初稿阶段需要鼓励；润色阶段需要犀利意见；取决于用户心态。

**实现要求**：
1. 提供三种 AI 反馈语气，一键切换：
   - **书童**：温和、鼓励、伴随式。适合写作初期。如："这个思路不错，要不要再展开一些？"
   - **诤友**：直接、客观、不客套。适合修改阶段。如："这段论证逻辑有跳跃，证据不够。"
   - **老学究**：引经据典、严谨、适度的疏离感。适合最后把关。如："此处文法虽通，然论据单薄，宜添一证。"
2. 所有 AI 输出内容相同，仅语气措辞不同

**示例**：
> 同一问题三种反馈：书童"这一段数据解读很棒！不过那个相关系数的描述可以再温和一些？" 诤友"r=0.32 你说是'强相关'？惯例 0.32 是中等偏弱，改一下。" 老学究"'强相关'二字犹言过矣。r=0.32 仅能解释一成方差，当以'中等程度相关'述之为宜。"

## 模块 H：发表与学术生涯

### I1. 报告准备

**功能目的**：帮助作者将长篇论文浓缩为不同时长的宣讲版本和展示材料。

**应用场景**：投稿后被接收需做口头报告；学术会议上做海报展示。

**实现要求**：
1. 论文导入，AI 自动生成：
   - **宣讲大纲**：3 分钟闪电演讲 / 15 分钟标准报告 / 30 分钟深度分享三个版本
   - **一页核心**：海报展示用——一张示意信息图 + 三句话核心发现 + 联系方式
   - **预判问答**：预测听众最可能问的 10 个问题及应答要点
2. 实现方式：LLM 摘要 + 结构生成

**示例**：
> 用户论文被顶会接收需做 15 分钟报告。AI 生成大纲：(1) 30 秒研究问题引入 (2) 2 分钟方法概述 (3) 5 分钟核心发现 (4) 3 分钟理论贡献与局限 (5) 预留 Q&A。

---

### I2. 方向拓展

**功能目的**：帮作者看清一篇论文在学术版图中的位置，从论文结束处发现新的研究方向。

**应用场景**：论文定稿后对"下一步做什么"感到迷茫。

**实现要求**：
1. 论文定稿后生成树状图：
   - 核心论点可延伸出哪些分支问题
   - 论文的局限性可演化为哪些新课题
2. 检索并关联近年顶会/顶刊的相关征稿主题，标注"此分支已有期刊征稿"
3. > **实现说明**：期刊征稿匹配依赖外部数据源，建议作为 Phase 3 后置功能。

## 优先级建议

### Phase 1：MVP（最小可行产品）

| 功能 | 理由 |
|------|------|
| 选题引导 | 低门槛、高感知，覆盖选题+思路展开两大核心痛点 |
